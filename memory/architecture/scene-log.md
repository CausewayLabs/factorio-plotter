# Feature Log: Scene

*Tracks planned features, open bugs, and completed work for the scene subsystem (entity model, rendering, viewport, persistence).*

## Foundation

- [x] (planned: 2026-05-23, completed: 2026-05-23) Scaffold Vite + TypeScript + React + Zustand project; `run.bat` at root (`npm run dev`), TS strict mode, subsystem dir layout. — decision: scaffolded manually (interactive `npm create vite` needs a TTY unavailable to the agent shell); identical output.
- [x] (planned: 2026-05-23, completed: 2026-05-23) Core TypeScript entity types: Bubble, Rail, Feeder, ParametricOrigin, ViewportTransform; Zustand store with authored slices (bubbles/rails maps) + derived feeders slot.
- [x] (planned: 2026-05-23, completed: 2026-05-23) `Rail.bubbleOrigin: string | null` — a rail can be anchored to a bubble's output (mutually exclusive with `parametricOrigin`); persisted, normalized on load (`bubbleOrigin ?? null`). First point resolves live to the bubble's output port via the consolidated `resolveRailPolyline(rail, rails, bubbles)`. — decision: separate nullable field rather than a discriminated-union `origin` rename, to keep the blast radius small; resolver is now the one place any origin is resolved (renderer/hit-test/solver all call it).

## Rendering

- [x] (planned: 2026-05-23, completed: 2026-05-23) SVG scene renderer with fixed draw order: rails → bubbles → feeders → missing-state badges.
- [x] (planned: 2026-05-23, completed: 2026-05-23) Bubble visual: label (truncated), output port, recipe-variant access via context menu, missing-requirement badge.
- [x] (planned: 2026-05-23, completed: 2026-05-23) Input ports redesigned as staggered left-side tabs: each input is a rounded rect showing the input's **full** product label (was last-word shorthand); boxes stack vertically and each successive box is wider so its left edge — the feeder attach point — sits further left, staggering the vertical feeder drops so they no longer overlap/pass through sibling ports. Satisfied = tinted in the resource color; unsatisfied = hollow with red dashed border. — decision: drove the layout from `bubbleInputBox` in geometry.ts, the single source of truth shared by the renderer and the solver, so feeders re-terminate on the tabs with zero solver changes. Promoted to Codebase Guide invariant.
- [x] (planned: 2026-05-23, completed: 2026-05-23) Input tabs snap to the side their feeder arrives from: each tab sits left or right of the bubble depending on whether its source is left/right of the bubble center (the "12:00" boundary). `bubbleInputBox` is now side-aware (`side: 'left'|'right'`, mirrored geometry, `port` = outer edge, `innerX` = edge toward bubble); each side stacks/staggers independently. — decision: the solver derives the side from the feeder direction and publishes a per-bubble `inputLayouts: InputSlot[]` into derived state; the renderer consumes it (falling back to all-left before the first solve). Keeps `bubbleInputBox` the geometry SoT while moving the side/index *decision* into the solver. Promoted to Codebase Guide invariant.
- [x] (planned: 2026-05-23, completed: 2026-05-23) Rail visual: per-resource color, distinct styling for supply vs. non-supply (private) state.
- [x] (planned: 2026-05-23, completed: 2026-05-23) Rails generalized to **buses** (`resourceTypes: string[]` + optional `label`): a rail carries N resource types; a plain rail is a bus of one. Color = the single resource's color, else a neutral bus color; label = `<name> (N)` / the type / `Bus (N)`. Legacy `resourceType` rails migrated on load (`normalizeRail`). — decision: deliberately supersedes the "one resource per rail" invariant to model Space Age recyclers (Fulgora scrap → ~12 items on one belt); the solver matches via `resourceTypes.includes(needed)` so the change is centralized.
- [x] (planned: 2026-05-23, completed: 2026-05-23) Feeder visual: non-interactive, visually distinct from rails (authored↔derived distinction).
- [x] (planned: 2026-05-23, completed: 2026-05-23) Lines drawn **behind** bubbles + **halo mask**: draw order is rails → feeders → bubbles, with rails+feeders wrapped in a `bubble-halo` SVG mask (white field minus `BUBBLE_RADIUS+15` black disks per bubble) so any line stops ~15px outside every bubble. — decision: a single userSpaceOnUse mask handles "own feeders and any pass-through lines" uniformly; pairs with direct feeders + bubble-center source attach so connections read as emanating from bubbles without colliding with their bodies.
- [x] (planned: 2026-05-28, completed: 2026-05-28) **Readability pass: font, rail-label knockout, uniform port width.** (a) Pinned a clean offline-safe UI font stack (`Inter` → `Segoe UI Variable` → `system-ui` …) via a `<style>` block in `index.html`, applied to both HTML chrome and SVG `<text>` (which otherwise falls back to the browser default serif). (b) Rail labels now render over a `CANVAS_BG` (#1a1a2e) knockout `rect` (width estimated from glyph count) so crossing rails/feeders no longer slice through the text; dropped the old stroke-halo in favor of the box. (c) Reverted the per-index input/output tab width **stagger** to a single `INPUT_BOX_WIDTH` (112) uniform per side — ports now line up across tabs. — decision: uniform over fit-to-content for now because fit would have to thread a label→width value through the solver's two-pass emit-port machinery (feeder/connector endpoints depend on the outer-edge port X); uniform is a zero-ripple revert and gives the upcoming feeder-overlap router a predictable aligned-port baseline. Updated the port-geometry invariant in Codebase Guide.
- [ ] (planned: 2026-05-28) **Feeder-drop de-overlap.** With uniform tab width, multiple feeders landing on the same side now share an X and can sit on top of one another. Route/offset them in the feeder solver so parallel drops are visually separable (the stagger used to mask this via tab width).
- [ ] (planned: 2026-05-28) **Fit-to-content port labels (kills truncation).** Make tab width follow its label (clamped min/max) instead of a fixed uniform width, removing "Fluoroketone …" / "Cryogenic Sci…" truncation. Requires publishing a per-slot `width` from the solver (label resolver in `SolverInput`) so geometry/renderer/hit-test consume one value and feeder endpoints stay on the tab edge. Pairs naturally with the feeder-de-overlap work.
- [x] (planned: 2026-05-28, completed: 2026-05-28) **Complete the color language (UI clarity #1).** Replaced the ~30-entry hand-maintained `RESOURCE_COLORS` map (which left every Space Age fluid/item — ammonia, water, fluoroketone hot/cold, lithium/holmium plate, brine, fluorine, cryo science — falling back to gray `#808080`) with a layered palette: authentic FactorioLab average-icon colors for all 896 items (`itemColors.json`, now emitted by `generate-bundled-recipes.mjs` from the icon defs' `color` field) as the base, the prior curated hues kept as overrides (preserves the ice-cyan / crude-oil-tan already on canvas and gives common items vivid, distinguishable colors), gray only for truly unknown names. Added a shared `ensureLegible` luminance-lift (generalized from the old label-only brightening) applied at two floors — lines/fills/strokes (96) and labels (170) — so dark authentic hues stay visible on the near-black canvas. — decision: funnel everything through `getResourceColor` (already consumed by rails, feeders, input/output tabs, and bubble strokes) so one base map lights up the whole graph; sourced colors from FactorioLab data rather than hand-maintaining, so coverage is complete and regenerable. Promoted to Codebase Guide invariant.
